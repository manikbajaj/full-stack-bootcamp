CSS Z-Index Demonstration

Z-Index: Controlling Stacking Order

z-index: 1
z-index: 2
z-index: 3

Explanation: The z-index property controls the stacking order of positioned elements. Elements with a higher z-index value are stacked on top of those with a lower z-index. Here, the green box (z-index: 3) is on top, followed by the red box (z-index: 2), and the blue box (z-index: 1) at the bottom.

Elements Without Z-Index

No z-index
No z-index

Explanation: When no z-index is specified, elements are stacked based on their order in the DOM. Here, the second box appears on top simply because it comes later in the code.

Stacking Context: Nested Elements

Parent Element (z-index: 10)
Child Element (z-index: 5)

Explanation: A new stacking context is created when a positioned element has a z-index. The child element’s z-index (5) is relative to the parent’s stacking context (z-index: 10), meaning the parent remains on top even though its z-index is higher.

Z-Index With Negative Values

z-index: -1
z-index: 1

Explanation: z-index can also be set to negative values, which places elements behind those with a positive z-index or no z-index. In this case, the blue box (z-index: -1) appears behind the red box (z-index: 1).
