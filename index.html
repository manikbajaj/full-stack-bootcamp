CSS Position Property Demonstration

Static Position (Default)

Static Position

Explanation: The default position of all HTML elements is static. In this mode, elements follow the normal document flow, meaning they appear one after another in the order they are written in the HTML. They do not respond to positioning properties like top, left, right, or bottom.

Relative Position

Relative Position

Explanation: With position: relative, the element is positioned relative to its original position in the normal document flow. The top and left properties adjust the element's position without removing it from the flow, meaning other elements still consider it in its original location.

Absolute Position

Absolute Position

Explanation: With position: absolute, the element is removed from the normal document flow and positioned relative to its closest positioned ancestor (an element with position: relative, absolute, or fixed). Here, it's positioned 100px from the top and left of the container.

Fixed Position

Fixed Position

Explanation: With position: fixed, the element is removed from the normal document flow and positioned relative to the viewport. It remains fixed in place as the page scrolls. In this case, the element stays fixed at 20px from the top and right of the viewport.

Sticky Position

Sticky Position

Explanation: With position: sticky, the element acts like relative until it reaches a specified scroll position, after which it "sticks" and behaves like fixed. In this case, the element will stick 10px from the top of the viewport as you scroll down.
